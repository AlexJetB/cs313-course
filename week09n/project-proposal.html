CS 313 | Web Engineering II

Project 2 Proposal

Overview

For the second half of the semester we will be building web applications using Node.js. As with Project 1, this will be an extended project that will drive the assignments and much of our learning for the rest of the semester.

Project Guidelines

The goal of this project is for you to gain experience in asynchronous callback functions, deepen your understanding of JavaScript programming, and understand the fundamentals of full-stack Web development.

With those goals in mind, the actual requirements of the project are fairly open-ended. The guidelines for the project are:

  1. Build something cool.

  2. Use Node.js.

  3. Demonstrate understanding of asynchronous callback functions.

  4. Handle authentication and persistence of state.

  5. Incorporate a data layer composed of either a DB (e.g., PostgreSQL) and/or third-party web services (e.g., Walmart, Twitter, FamilySearch, etc.).

You are welcome to implement this as a single page application (SPA) or using multiple pages generated with a template view engine, such as EJS.

As an example, consider an educational Web application that teaches Math or computer science principles via a game or interactive programming site. It could allow different users to log in, and track their progress through the learning modules.

Proposal Guidelines

Your proposal document need not be long, but it should outline the general idea of your project and how it will meet the necessary requirements. Please include the following sections:

  1. Overall project description. Describe the overall application.

  2. Pages. Describe the layout of your application. Will it be a single page application? If so, describe the key elements of the page and how the user will interact with them. Will it include multiple pages? If so, list the pages you anticipate and the interaction among them. Simple mockups may be helpful in illustrating your idea.

  3. Data layer. Describe the necessary entities that you will store in the database, or the 3rd party API that you plan to use.

  4. Milestones. Your weekly assignments for the next four weeks will be portions of your project, as follows:

    • Week 10: Milestone 1

    • Week 11: Milestone 2

    • Week 12: Milestone 3

    • Week 13: Milestone 4 / Project Demo

    For each of these milestones, indicate the portions of the project you will have done at that point. In order to prepare for deployment issues that inevitably arise, for your Milestone 3 (Week 12), please include "Have a version of the application working on our server" along with any other plans you have.
