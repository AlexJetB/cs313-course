04 Teach : Team Activity

Special Instructions for Online Students

You need to coordinate a one hour synchronous meeting with your group for this activity.

Overview

Design a simple relational database structure and create the schema in a Postgres database.

Instructions

Consider that you have been asked to write an application to store notes for users for general conference talks. The basic requirements are:

  1. This system must track notes for multiple users.

  2. The speakers should be tracked separately from the conference because one speaker will speak at multiple conferences.

  3. Individual sessions of conference (e.g., Saturday morning, Sunday afternoon) need not be tracked, rather simply the year and month, (e.g., April 2009, Oct 2012).

  4. You need not account for the fact that some speakers will speak multiple times at a conference, but assume that a note can be tied to a "speaker + conference."

  5. Multiple notes can be added by the same user about a single conference talk.

Your assignment is to create the database for this application. Please take into account good database design principles such as the elimination of redundant data, data type selection, primary keys, and foreign key relationships.

Core Requirements

  1. Design your database visually.

    The first step is to put together a design for the database (i.e., outside of your PostgreSQL environment). You may want to use a collaborative image drawing or document editing application for this. It doesn't have to be formal or follow exact UML, but rather be a tool to discuss the things to include.

  2. Log into PostreSQL on one of your computers and create the tables you have designed. Make sure that everyone on the team has a way of seeing and contributing to the SQL that is written.

    It is often easiest to write the commands in a text editor first, and then copy and paste it into your SQL window. That way if (when!) you get errors, it's much easier to correct them and paste the content in again.

  3. Add appropriate foreign key relationships for the tables you created.

Stretch Challenges

After finishing the core requirements, ensure that everyone is at that point and understands the material. When everyone has completed the core requirements, you can move on to these stretch challenges.

  1. Change your database so that it tracks each session separately (Saturday Morning vs. Saturday Afternoon). Ensure that this also allows for different talks by the same speaker to be treated separately.

  2. Create your database at Heroku, using one of your accounts.

  3. Practice inserting some data from the last general conference. Insert at least 3 talks, and at least 2 notes per talk. Then demonstrate that you can query to receive all the notes about a particular talk.
