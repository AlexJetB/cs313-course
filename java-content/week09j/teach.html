CS 313 | Web Engineering II

09 Teach : Team Activity

Special Instructions for Online Students

You need to coordinate a one hour synchronous meeting with your group for this activity.

Objective

Be able to use servlets and JSPs together to accomplish a simple task.

Overview

Create a login page (JSP) that accepts a username and password. If a correct username and password is entered, it should direct the user to a welcome page showing their username. If an incorrect username and password are entered, it should direct them to an incorrect password page.

Instructions

First, create a JSP login page with a username and password box. Submission of the form should post to a login servlet.

  1. Create a login servlet to check for a valid username and password (at this point a correct username and password to check against can be hard coded into the servlet's code). If the username and password match, put the username on the session and send the user to a welcome page, if not send them to an incorrect login page.

    This also requires that you create a JSP incorrect login page.

  2. Create a JSP welcome page that says “Welcome [username]” (getting the username from the session). Also have a link to logout, this should post to a logout servlet.

  3. Create a logout servlet that removes the username from the session if present, and redirects the user back to the login page.

Stretch Challenges

After finishing the core requirements, ensure that everyone is at that point and understands the material. When everyone has completed the core requirements, you can move on to these stretch challenges.

  1. Combine the incorrect login page with the sign in page. So that it is only one page, but the presence of extra data causes it to display an incorrect login message before the username and password boxes. This can be done by having the servlet store information on the request before transferring to the JSP.

  2. Deploy your app to one of your servers.

  3. Store a list of usernames and passwords in a file on the server and use that to validate in your login servlet.
