CS 313 | Web Engineering II

11 Teach : Team Activity

Special Instructions for Online Students

You need to coordinate a one hour synchronous meeting with your group for this activity.

Overview

For this activity, we will be using JDBC to connect to a PostgreSQL database and display information in a Java web application.

Consider the case of a list of ancestors in a database. For simplicity, each person should have a first name, last name, and birthday. For a stretch challenge you will create a relationship between parents and children modeled in the database (many to many).

Then on a web page, you should display a list of all the people in the database. Each name should be a link, where if you click on it, a person details page should be display that shows the person, along with their children (if they exist in the database) and their parents (if they exist in the database).

Instructions

Core requirements

  1. First, ensure that you have PostgreSQL installed on your development machine. Then, create a Person table in the database. Finally, insert some test people.

  2. Create a Java web application with a page to display each person in the database.

    • Don't forget to include the dependcies in your pom.xml (or the JAR's) for the JDBC connector, as well as the JSTL.

    • Write a simple query to select all the people from the database and return them.

  3. Make each name on this page a link to a Person Details page. At this point, have the Person Details page simply display the name and birthday of the person.

Stretch Challenges

After finishing the core requirements, ensure that everyone is at that point and understands the material. When everyone has completed the core requirements, you can move on to these stretch challenges.

  1. Create a ChildOf table in the database to model the many-to-many relationship.

  2. On the Person Details page, display the children and ancestors of the current person.

  3. Convert the names of the children and ancestor into links that go to the person details page for those people.
