Before meeting with your team, please review the following material on database access using Java. Please note that in order to interact with a database, you need to have a Postgres instance that you can interact with. Thus, while you are developing your code you need to do one of the following:

  1. Deploy your development code to your server where your Postgres instance exists.

  2. Install a local instance of Postgres for development

  3. Open the Postgres port on your server to allow remote connections, so that you can connect directly to it from your local machine.

Option 1 (deploying your code for each test), can be cumbersome, especially if you are not using an automated build/deployment process.

Option 3 can be a major security risk. It is best practice to NOT allow direct connection to your database from outside your server(or data center/VPN, if your application is not all running on the same server).

Thus, the best approach is to install Postgres locally on your development machine, so you can test and debug code there, and then when it is ready to deploy, you actually deploy it to your server and do final testing there.

As noted in the tutorial, you will need to get the JDBC driver for your database (in our case Postgres). This can be downloaded directly from this page. This JAR should be put into your lib directory.

Alternatively, if you are using Maven, you can simply add this dependency to your pom.xml file, instead of manually downloading the JAR file: 


<dependency>
  <groupId>org.postgresql</groupId>
  <artifactId>postgresql</artifactId>
  <version>9.4.1212</version>
</dependency>
