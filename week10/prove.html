CS 313 | Web Engineering II

10 Prove : Assignment - Milestone 1 - End to end

Overview

Successfully get a request go from the UI to the data layer and back, for one of the components of your project.

Instructions

For this week's assignment, you will be working on your individual project. Your task is to complete one piece of your project "from end to end." In other words, starting from the user interface, the user does something that causes a request to be made to the server, which queries your data layer (e.g., your database, a third-party API, etc.), returns that information back in a response, which is handled by client-side code in the the browser to render a result to the user.

For example, consider an application that makes use of restaurants in a certain area. For this week's assignment, you might complete the following scenario:

  1. The user clicks a button to retrieve a list of restaurants.

  2. In client-side JavaScript, this triggers a background, AJAX request to a /getRestaurants URL on the server.

  3. The server handles this request and queries the database to select all the records from the restaurant table.

  4. These results are sent in JSON format in a response back to the client.

  5. Client-side JavaScript receives the result and updates the DOM (i.e., the web page the user is viewing) to show the results in a bulleted list.

Getting this single, "thin thread" complete from end to end can be challenging, because it requires so many individual pieces to work together. However, once you have this in place, the rest of the components of your project should follow along similar lines.

Instructor Tips

When you are going end-to-end like this from the web service to the database and back, there are a ton of things that can go wrong! I always try to get the simplest possible components working in isolation first, then expand to the actual data I need. For example, while working on the AJAX request, I would have the server return a hard-coded result of "dummy data" first, and make sure that is working. Then, I would expand to use the actual database.

As you get deeper into your project, it is likely that you will need to make changes to what you originally planned. This is natural and expected. For this and future milestone submissions, you should do what is best for your project, even if that is different than what you originally designed.
