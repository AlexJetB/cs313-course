Web Services

This week, we will be talk about APIs and in particular, Web services. In short, Application Programming Interfaces (APIs) define an interface for one program to talk to another. In our case, we are particularly interested in Web services, where this communication happens over the Web (see also, Stack Overflow: What's the difference between a Web service and an API).

Please read the following:

  1. The introduction to the Wikipedia article on Rest. While there are other types of Web services, the most popular is REST, which takes advantage of the HTTP standard for stateless communication, and this introduction gives a good overview of the concept

  2. RESTful Web Services: The Basics. This article gives an overview of how this approach is used and some best practices around creating Web services. The correct use of the HTTP verbs is summarized in this table from the above mentioned Wikipedia article.

In addition, here are a few helpful references:

Accessing PostgreSQL from Node.js

The pg module for Node.js contains the functions to interact with PostgreSQL from Node.js. Once you get the hang of making the connection and getting results back, the actual queries are the sames as we used in the PHP section of the course.

Please read the following:

  1. node-postgres. Please read over the Welcome, Connecting, Queries, and Pooling sections of this tutorial to see examples of code to make connections. At this point, we will focus on using the Callback notation.

    The npm documentation for node-postgres gives some additional connection examples.

Using node-postgres to Create Endpoints

Putting it all together, we can now use node-postgres in combination with Express.js to create a Web service to return data from our database.

  1. Database Endpoints in Node.js gives an example of creating Web service endpoints for getting and inserting some users.
