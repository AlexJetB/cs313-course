CS 313 | Web Engineering II

02 Teach : Team Activity

Overview

For this assignment we will be practicing PHP fundamentals.

Instructions

With your team, complete the following assignment. You will create only one page, but each of you should be actively engaged in writing the code and contributing to the solution. Then, at the end of the activity make sure that everyone on the team gets a copy of the code that you wrote. Each week, we will switch to let someone else be the "main driver" of the keyboard.

Core Requirements

  1. Create a PHP web application with three pages:

    • about-us.php - Come up with a topic for a fictitious company, and create an "About Us" page that describes your company.

    • home.php - This page should welcome the user to your site. Later it will be updated to include the user's login name, but for now, it can just say, "Welcome. You are not logged in."

    • login.php - This page should have two links or buttons on it: "Log in as Administrator", "Log in as Tester".

    Verify that you can load each of your pages in a browser using the "localhost" URL.

  2. Add a common header / navbar for all of your pages. It should display your fictitious company's name, and have links to the three pages. Create this header as a separate PHP file that can be included by each of the three pages.

  3. Change the navbar so that the current page is highlighted in some way (e.g., when you are on the about page, the background color for the "about" link is different.).

Stretch Challenges

Please make sure everyone on your team is caught up with the concepts of the core requirements, before moving onto the stretch challenges.

  1. Make it so that the login buttons you created on the login page store the name "Administrator" or "Tester" as a session variable. This may require you to create an additional PHP script(s) to handle the login. Don't worry about making it a form with passwords or anything at this point.

    After logging the user in, have the web application automatically redirect them to the home page. Then update the page to display their username (i.e., "Administrator" or "Tester".

  2. Add a Logout button or link to the home page. This should remove any session variables.

    Update the home page to automatically redirect the user to the login page if no user is logged in.

  3. Deploy this site to one of your Heroku accounts.

Instructor's Solution

As a part of this team activity, you are expected to look over a solution from the instructor, to compare your approach to that one. One of the questions on the I-Learn submission will ask you to provide insights from this comparison.

Even if you use the instructor's code to help you, you are welcome to report that you finished the core requirements, if you code them up yourself.
