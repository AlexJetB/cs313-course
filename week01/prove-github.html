01 Prove : Hello World Assignment : Pushing Your Code to GitHub

GitHub

In this document we will set up a repository at GitHub, and push your source code there as well.

GitHub is essentially a big database of source code for people all around the world. You can make your source code public or private and configure many different permissions and settings. Each project has its own repository, or "repo". This repository will be a clone of the one on your local computer. Thus, you will have three copies of your code, 1) on your local computer, 2) at Heroku, and 3) at GitHub.

Create a new repository

  1. Go to GitHub.com and sign up for a new account if you haven't already. If you already have an account, that is perfect, you can use it here.

  2. After creating an account, return to GitHub.com and sign in. In the upper right corner of the screen you should see a "+" icon. Click this and select "New Repository".

  3. Choose a name for your repository (likely cs313-php or something similar).

  4. Leave the repo public and do not check the box for a readme. Leave .gitignore and license as "none".

    Instructor Tip:

    It's not technically a problem if you add these other files now, but it makes it a little more difficult when you go to push to GitHub for the first time, because you'll have conflicts that you'll have to resolve. So at this point, it's easiest to leave everything blank.

  5. Click the button to create the repository.

  6. On the page that follows, you should see a URL for your new repository (e.g., "https://github.com/sburton42/cs313-php-17w.git"). Copy this URL for use later on.

Connect your code to your repository

We now have an empty repository at GitHub, so we can now connect our local copy to it.

  1. In a terminal window, change to the directory of your PHP project. For example:

    
$ cd cs313-php

  2. First, we can see all the remotes that our local repository is connected to. This is done by running the following command:

    
$ git remote -v
heroku  https://git.heroku.com/secret-sierra-28842.git (fetch)
heroku  https://git.heroku.com/secret-sierra-28842.git (push)
origin  https://github.com/byui-cs/cs313-php (fetch)
origin  https://github.com/byui-cs/cs313-php (push)

    In my case, I have two remotes, one called "origin" where I cloned from originally, and the second called, "heroku" that we use to push to our Heroku server. We will now add a third remote for GitHub.

    A remote simply means this is another repository that we may sync up with. When we want to push or pull from any of these repositories we can use the remote name (e.g., "heroku") rather than having to type out the complete URL.

  3. Add a remote for GitHub, pasting in the URL that you copied earlier. This is done as follows, but of course replacing the URL with your own.

    
$ git remote add github https://github.com/sburton42/cs313-php-17w.git

  4. Verify that your new remote has been added, by running the command as before to list the remotes:

    
$ git remote -v
github  https://github.com/sburton42/cs313-php-17w.git (fetch)
github  https://github.com/sburton42/cs313-php-17w.git (push)
heroku  https://git.heroku.com/secret-sierra-28842.git (fetch)
heroku  https://git.heroku.com/secret-sierra-28842.git (push)
origin  https://github.com/byui-cs/cs313-php (fetch)
origin  https://github.com/byui-cs/cs313-php (push)

    In the above output, you can see that "github" has been added as a remote.

  5. We can now push our source code to GitHub, by pushing our repository to the "github" remote. We do this the same way we pushed to Heroku, except that we specify that we want the github remote this time as follows:

    
$ git push github master

  6. The final step is to return to your repository page in the browser (at GitHub.com) and refresh the page. You should now see your code!
